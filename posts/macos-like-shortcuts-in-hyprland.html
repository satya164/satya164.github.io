Copy & Paste in Terminal

It's great that copy and paste use the same shortcuts as in other applications on macOS. However, since Linux uses Ctrl + C for copy, Terminal apps need to use a different shortcut as Ctrl + C is already used for killing processes in terminals. So our remapping for copy and paste won't work.

Luckily, we can also add custom logic for our keybindings, where we can check whether the active window is a terminal or not using hyprctl. Then we can dispatch Ctrl + Shift + C and Ctrl + Shift + V instead.
